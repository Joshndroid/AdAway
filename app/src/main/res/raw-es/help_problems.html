Copy of hosts file failed on Android 9+

Latest versions of Android use read-only /system partition.
If you are using Magisk as root solution, ensure you enable the systemless hosts module then reboot.

No hay suficiente espacio en la partición

Intente cambiar el archivo de hosts de destino en preferencias a /data/data/hosts (o /data/hosts) y aplicar AdAway nuevamente.

No funciona sobre Android 4.4+

Intente cambiar el archivo de hosts de destino en las preferencias de /data/data/hosts a /data/hosts o /system/etc/hosts y vuelva a aplicar AdAway.

¡No bloquea anuncios en la aplicación XYZ!

Es posible que falten algunos nombres de host en los archivos de hosts proporcionados por las Fuentes de Hosts o la aplicación ha agrupado las imágenes para proporcionar anuncios sin acceder a Internet.

Puede registrar solicitudes de DNS (Menú->Registrar solicitudes de DNS) desde AdAway para averiguar qué nombres de host adicionales deben bloquearse.

Agregue los nombres de host sospechosos a su propia Lista negra presionando las entradas en el Registro e informe estos nombres de host cuando los haya verificado en la Bandeja de entrada de hosts del foro de hosts-file.net.

Anuncios en caché

A veces las aplicaciones almacenan anuncios en caché después de la descarga. Esto lleva a los anuncios restantes en algunas aplicaciones. Puede intentar eliminar la memoria caché para estas aplicaciones en la lista de aplicaciones de Android para evitar este problema.

¡La aplicación XYZ ya no funciona!

Algunas aplicaciones deben comunicarse con un nombre de host bloqueado por AdAway o negarse a trabajar cuando no se puede acceder a los nombres de host que deberían servir anuncios. Visite https://github.com/AdAway/AdAway/wiki/ProblematicApps para obtener una lista de aplicaciones conocidas que tienen problemas. De lo contrario, averigüe qué nombres de host se necesitan y agréguelos a su lista blanca en sus listas e informe de ellos al rastreador de errores de AdAway.

El botón "Atrás" en los navegadores web no está funcionando

Puede habilitar el servidor local de AdAway en las preferencias como solución provisional.
